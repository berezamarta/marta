Git: Основи та можливості

Коротка інформація

Системи контролю версій — це інструменти, які дозволяють відстежувати зміни у файлах і координувати роботу кількох людей над проектом. Git — одна з найпопулярніших розподілених систем контролю версій, створена Лінусом Торвальдсом у 2005 році. Вона використовується для керування кодом у проектах будь-якого масштабу.

Основні можливості та переваги Git

  • Розподілена система — кожен користувач має повну копію репозиторію.
  • Висока швидкість роботи.
  • Гнучке гілкування та злиття.
  • Відстеження історії змін.
  • Зручна співпраця у команді.
  • Відкритий код і велика спільнота.

10 базових команд Git

  1. git init — створює новий Git-репозиторій.
  2. git clone URL — копіює віддалений репозиторій.
  3. git status — показує стан файлів.
  4. git add filename — додає файл у staging area.
  5. git commit -m "повідомлення" — зберігає зміни.
  6. git log — перегляд історії комітів.
  7. git branch — показує/створює гілки.
  8. git checkout branch_name — перехід між гілками.
  9. git merge branch_name — злиття гілки.
  10. git push origin main — відправка змін у віддалений репозиторій.
